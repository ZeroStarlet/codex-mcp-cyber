# 强制送审（关键面）

满足下表**任一原则**的改动 → **强制送审**，即使改动很小，**不得 Claude 自我放行**。PROMPT 的「本次重点」须标注高风险面；复审到 ✅ PASS 才合入。

| 原则 | 说明 | 跨领域举例 |
|------|------|-----------|
| **不可逆** | 错了没法撤销或回滚代价极高 | 数据库迁移（DDL）、文件删除 / 覆盖写入、生产配置变更、固件烧录、远程部署 |
| **信任边界** | 处理来自本模块之外的数据，是注入 / 投毒入口 | 用户输入解析、网络反序列化、文件读取 / 导入、IPC、传感器数据、外部 API 响应 |
| **静默失效** | 错了不 crash，结果悄悄变错 | 数值精度、权限判定、配置覆盖 / 合并、缓存键冲突、浮点比较、正则回溯 |
| **级联影响** | 错了会扩散到其他模块 / 系统 / 消费者 | 公共库 API 签名、共享类型 / 协议、消息 schema、微服务契约、插件接口 |
| **密钥与加密材料** | 密钥 / 签名 / 哈希选型出错导致安全性崩塌 | 密码哈希、Token 签发、证书校验、算法选型、随机数、密钥派生 |

适用于系统编程、游戏、数据管道、嵌入式、CLI、Web、移动端等任意领域。
